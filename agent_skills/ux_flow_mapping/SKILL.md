# UX Flow Mapping

## Working Style

- Identify how the user enters the flow, what decision or action they take, and how the system responds.
- Include empty, loading, and failure states when they affect the experience.
- Keep the user goal visible while making implementation choices.

## Output Expectations

- Make the flow legible to both design and engineering readers.
- If the flow is incomplete, note which states or transitions are still undefined.
