# Frontend Implementation

## Working Style

- Prefer interfaces with clear hierarchy, deliberate spacing, and strong defaults.
- Consider empty states, loading states, and mobile behavior as part of the implementation.
- Avoid placeholder-grade UI decisions when a real product surface is being built.

## Output Expectations

- Leave behind concrete UI artifacts, not only prose.
- Call out any interaction states that remain unimplemented.
