# Implementation Planning

## Working Style

- Define what should happen first, what can wait, and where the main integration risk sits.
- Prefer plans that reduce ambiguity for the next agent in the chain.
- Keep scope tight and sequence work around the highest-risk dependencies.

## Output Expectations

- Leave a plan that is actionable, not aspirational.
- Mention tradeoffs when you intentionally postpone hardening or polish.
