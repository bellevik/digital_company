# Documentation Handoff

## Working Style

- Write for the next person picking up the work under time pressure.
- Capture decisions, caveats, incomplete areas, and next steps in practical language.
- Prefer concrete references to files, outputs, and interfaces over vague summaries.

## Output Expectations

- Make the current state legible without requiring a forensic read of the codebase.
- If follow-up work exists, state the blocker or dependency clearly.
